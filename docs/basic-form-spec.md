### A. 色の言語（全モード共通・認知負荷最小が最上位目標）

「色が増える＝認知処理が重い」。デフォルトは最小色：

| 役割 | 色 | 備考 |
|---|---|---|
| 押す音（chord モードのコード構成音 / input モードの押下音）| **明るいグレー**（`--pad-basic-chord` を grey 系に） | デフォルト。Push では LED 色変更可（既存 push*Color 設定） |
| スケール音 | **青** `--pad-basic-scale` | 背景 |
| スケールのルート（トニック = `AppState.key`）| **オレンジ** `--pad-root` | 常設アンカー |
| オフ | **濃いグレー** `--pad-off` | |
| 度数/特性音の色分け（3rd/7th/tension/特性音）| **Setting で opt-in**（既存 colorOff トグル） | 元は読字障害アクセシビリティ用。大半は負荷高すぎ→既定 OFF。モード学習（ルート+特性音）で価値。トグル名は「コード/スケールを色分け」等、度数色も含む実態に広げる |

- **押す音とスケールルートは別色**（グレー ≠ オレンジ）。スケール（青）とも別。
- input モードも**押下音は単色グレー**（自由入力→判定、キー非依存。スケール背景やトニックは出さない＝従来パラダイム維持）。

### B. スケール背景は常設

chord モード（基本形・全ポジション表示の両方）と scale モードで、**青スケール + オレンジのトニックを常に背景表示**。その上に押す音（グレー）が乗る。

### C. 基本形（chord モード既定 = showAllPositions=false）

- スケール背景（青+オレンジトニック）+ コードの **1 配置だけ**（グレー）。
- **ナビは画面で見える操作が主**（goryugo「みんなショートカット覚えられない」）。ショートカットは覚えてる人へのおまけ。
  - **上下 = 転回（オクターブ越え可）**：基本は転回。ただし**現状の転回 0-3 の頭打ちを外し、最上転回の先＝次のオクターブの転回へ続いて連続で登れる**ようにする（下は逆＝降りる）。画面の上下ボタンが主操作。
  - **Shift+上 = オクターブ上 / Shift+下 = オクターブ下**（ショートカット。覚えてる人向けの粗い移動）。
  - **スペース（+ 画面アフォーダンス）= 同 register の別配置**（同じ音・別パッド）。横の切替。**残す**（上下に吸収すると4和音で手数過多）。
- **他ポジションの合図 = 「1/3」式の数字バッジ（点滅は廃止）**。同 register の別配置が2つ以上ある時、「1/3」（現在/総数）を表示。点滅は負荷高いので使わない。

### D. 全ポジション表示（showAllPositions=true）

- スケール背景（青+オレンジトニック）+ コード全配置（グレー）+ **ABCD ラベル（破線枠なし）** を各配置のルート（R）に付ける。配置に複数候補があれば「A1/3」等。
- **破線枠は廃止**（重い）。ボックス選択の位置保持ロジックも使わない → **#13 バグは原理的に消える**。

### E. これは別タスク（今回の実装に含めない）

- ダイアトニックバーからコード選択時に発音する（タスク化済み）。

### F. 現状コードとの主な差分（実装対象）

1. 押す音 白(#e8ecf0) → **グレー**（`--pad-basic-chord` を grey に。input 押下音も単色グレーに）
2. **点滅 → 「1/3」バッジ**（`basicFormBlink` / `.basic-alt-blink` 廃止、バッジ描画へ）
3. **上下 = 転回（オクターブ越え可）**：現 `setInversion`(0-3) の頭打ちを外し、オクターブを越えて連続で上下できるようにする（転回の概念は維持、ただし循環で止めず次オクターブへ続ける）。**Shift+上下 = オクターブジャンプ**（現 `shiftBasicFormOctave` を維持）。SPACE の同register別配置は維持
4. **全ポジション表示にスケール背景 + ABCD ラベル(破線なし)** を追加（現状=グレー全配置のみ）
5. colorOff トグルの**ラベル変更**（度数色も含む実態に）
6. 凡例の押す音スウォッチをグレーに（基本形凡例は既に対応済）

## 1. 用語（取り違え防止）

| 用語 | 定義 |
|---|---|
| **基本形** (basic form) | コードのコンパクトな 1 配置。`calcAllVoicingPositions` のコンパクト順 [0]。 |
| **他のポジション** (other positions) | **同じ音（＝同じ音域・同じピッチ）を、別のパッド配置で押さえること**。パッド上は同じ音名が複数箇所に出るため、同じ和音を別の場所で握れる。= `calcAllVoicingPositions` の alternatives。**オクターブ違いではない**（ここが過去の取り違え）。 |
| **オクターブ移動** | 音域そのものを上下する操作。**他のポジションとは別軸**。Shift+上下に割当。 |
| **スケール背景** | 現在のキーのスケール音を薄く表示した背景レイヤー（`V2.3 Scale Overlay` の常時版）。 |

## 2. コードモード基本形デフォルトの表示（showAllPositions = false）

認知負荷を下げるため、コードモードの既定では「押さえる形を 1 つだけ」明確に出す。

1. **背景: スケールを薄く表示（音名が読める）**
   - 現在のキー（`AppState.key` + `AppState.scaleIdx`）のスケール音を薄い色で表示。
   - **音名は読めること**（塗りつぶしで潰さない。文字は明るめ）。
   - **スケールのルートは位置が分かる**（他のスケール音より少し明るく＝トニックのアンカー）。ただし**コードトーンと重なっている場合は不要**（既にコードとして見えているため）。
2. **前面: コードを単色 1 色で重ねる**
   - **ルートもコードトーンも同じ色**（ルートだけ別色＝オレンジにしない）。
   - スケール背景とは**別の色**で、目立つこと。
   - **1 ポジション（基本形）だけ**を出す。全配置を一度に出さない。
3. **他のポジションがある時は点滅で知らせる**
   - 基本形の形（コードのパッド）を**控えめに点滅**させ、「他にも押さえ方がある」と示す。点滅は疲れない程度にゆっくり・浅く。
   - = `Phase 4.9` の「循環可能バッジ脈動」と同じ思想を、バッジでなくパッドに適用したもの。
4. **ナビゲーション**
   - **スペース** = 次の他ポジション（同音域・別配置）へ巡回（+その音を発音。音が出てよい）。1 配置しかなければ従来通り再生。
   - **Shift+上 / Shift+下** = オクターブ上 / 下（音域を動かす）。グリッドに収まる間は形だけ動かし、収まらなければグリッドのオクターブ範囲ごと連動して見切れを防ぐ。
   - **上 / 下** = 転回（既存）。**左 / 右** = 半音移動（既存）。

### 2.1 画面に出してよいもの・出してはいけないもの（うりなみさん 2026-05-28 明示）

- **画面に出るのは基本形の 1 配置だけ**。他のポジションは**画面に描かない**（白枠・カッコ・ゴースト・薄い別配置のいずれも出さない）。他のポジションは**スペースで切り替えて初めて見える**。一度に複数配置を見せると認知負荷が増える。
- **白いカッコ（白枠）はどこにも出さない**。基本形のコードパッドも**塗り（solid 色）だけで枠線なし**。
  - 実装: 基本形時は全パッド `stroke: none`（render.js の `basicPadSet` 分岐）。半透明 fill の縁が枠線に見える錯覚を避けるため、スケール背景は**縁が目立たない程度に solid な青**にする（`--pad-overlay` の rgba 0.2 は透けすぎ → 縁の錯覚が出る。もう少し不透明な青を使う）。
- 点滅は「他のポジションあり」の合図として**基本形の形そのもの**に控えめにかける（別配置を描いて点滅させるのではない）。

### 2.2 色の制約（うりなみさん 2026-05-28 明示）

- **コードの色は、スケールのルートの色と同じにしてはいけない**（混乱が生じる）。コード = 単色 1 色、スケールルート = それとは別の色。
- **スケールの色は青**（うりなみさん指定の青系）。スケールルートはその青の中で識別できる明るさ。
- コード = 青以外の単色（暫定オレンジ）。スケールルート（青系）と明確に別色であること。

## 3. 全ポジション表示トグル（showAllPositions = true）

- **配置**: 画面（コードビルダーの Voicing 欄）＋ 表示設定モーダル の 2 箇所。`AppState.showAllPositions`（既定 false、localStorage 永続化）。

- **色**: 全ポジション表示では**ルートと構成音は別色**（俯瞰＝詳細ビューなのでそれでよい。基本形は単色で簡素、全ポジションは色分けで詳細、という役割分担）。

## 5. 内部状態・実装の要点

- **`VoicingState.basicPosIdx`**: 現在表示中の同音域・別配置の index。`calcAllVoicingPositions` 結果の何番目か。view-only、**serialize しない**。コード変更・転回・半音移動・キー変更で 0 にリセット（`resetVoicingSelection()` ＋ 半音移動ハンドラ）。
- **`VoicingState.basicOctave`**: Shift+上下のオクターブオフセット。view-only、非 serialize。毎 render でグリッドに収まるよう clamp。
- **`basicFormArrangements()`** (theory.js): 現在のコードの同音域・別配置を `calcAllVoicingPositions` で列挙（bass の全グリッド位置を起点に集めて重複除去・コンパクト順）。[0] = 基本形。
- **`cycleBasicFormPosition()`** (theory.js): スペースで `basicPosIdx` を次へ。同じピッチのまま配置だけ変わる（発音は同じ和音）。
- **`chordBasicFormActive()`** (render.js): 基本形デフォルト表示の発動条件。
- **render**: 基本形時は `basicFormArrangements()[basicPosIdx]` のパッド集合を `basicFormPadSet` として渡し、レイヤリング（スケール薄背景＋単色コード）で描画。点滅は配置が 2 つ以上ある時のみ。

### 5.1 #13（半音移動バグ）回避

基本形は **`VoicingState.selectedBoxIdx` / `_preservePosition` / `lastBoxes` を一切触らない**（毎 render で計算）。これにより半音移動時のボックス位置保持ロジック（`computeAndDrawVoicingBoxes` の transpose 形状マッチ）に入らず、pre-existing の #13 を回避する。全ポジション表示 ON の時だけボックス機構が動く。

### 5.2 ゲート条件（chordBasicFormActive が true になる条件）

すべて満たす時のみ基本形デフォルト:
- `AppState.mode === 'chord'`
- `!AppState.showAllPositions`
- `!AppState.padCFixed`
- `BuilderState.root !== null && BuilderState.quality`
- `!TastyState.enabled && !StockState.enabled`
- ギターエンジン非アクティブ
- `!(_voicingReflectMode && _guitarSyncSource === 'position') && !_stockReflectMode`
- **`!_instrumentMidiSet && !_instrumentPadSet`**（楽器入力 WYSIWYG 表示中は基本形を出さず従来表示に譲る＝V2.7/V2.8 統合を壊さない。監査 BLOCKER B-1 対応）
- **`padExtNotes.size === 0`**（パッドクリックで追加した音が builder に未反映の間は全表示）
- `VoicingState.selectedBoxIdx === null`
- グリッドは 64 パッド（32 パッド `grid` 引数時は適用しない）

## 6. 確定事項 / 調整可能事項

**確定（うりなみさん明言）**:
- 認知負荷を増やさない＝基本形 1 つ＋必要時だけ巡回。
- 他のポジション = 同音域・別配置（オクターブ違いではない）。スペースで巡回。
- スケールを薄く背景表示・音名が読める・ルートが分かる。
- ルートとコードトーンは同じ色。
- スペース切替で音が出てよい。
- 全ポジション表示はルートと構成音が別色のまま。

**調整可能（実物を見て決める）**:
- コードの色（暫定オレンジ）、スケール背景の濃さ（暫定 `--pad-overlay`）、トニックの見分け方（暫定 `--pad-overlay-hi`）。
- 点滅の速さ・浅さ（暫定 2.6s 周期・opacity 1↔0.78）。

## 7. 検証観点

- 基本形: コード確定で 1 配置のみ単色点灯、スケール薄背景の音名が読める、トニック識別可能、黒い穴・余分な白枠なし。
- スペース: `basicPosIdx` が巡回し、**ピッチは不変・パッド配置だけ変わる**（同音域）。発音される。配置 1 つなら巡回しない。
- 点滅: 配置 2 つ以上の時だけ点滅。
- Shift+上下: 音域が動く（収まる間は形のみ、端でグリッド範囲連動）。
- 全ポジション表示トグル ON/OFF: 俯瞰+ボックス ↔ 基本形。
- #13: 基本形で半音移動を繰り返しても `selectedBoxIdx` null・`_preservePosition` false のまま崩れない。
- 回帰なし: TASTY / STOCK / Guitar / scale / 32-pad / padCFixed / 楽器入力×Chord。
- `npm test` + 9 言語 i18n + Codex(or Claude フォールバック)監査。
